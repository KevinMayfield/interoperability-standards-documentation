Reference Policy

Interaction summary
  • SHALL support update, read, search-type.

Documentation

The update expects an identifier parameter i.e. PUT [base]/Encounter?identifier={system|}[code].

Example:

PUT [base]/Encounter?identifier=https://fhir.leedsth.nhs.uk/YrVisit|YOR123456

This is the same upsert behaviour as described IHE PIXm around the Patient resource.
